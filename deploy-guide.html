GitHubリポジトリの準備

1-1
バックエンド用リポジトリを作成する ▶

① github.com にログイン → 右上の「+」→「New repository」

② 以下の設定で作成する:

項目値
Repository namebuaizai-backend
VisibilityPrivate（推奨）
Initialize with READMEチェックしない

③ ローカルPCで buaizai-backend/ フォルダに移動してpush:

cd buaizai-backend
git init
git add .
git commit -m "feat: initial backend"
git branch -M main
git remote add origin https://github.com/あなたのID/buaizai-backend.git
git push -u origin main

3-2
環境変数（Environment Variables）を設定する ▶

Web Service設定画面 → 「Environment」タブ → 以下を1行ずつ追加:

KeyValue（内容）必須
DATABASE_URLStep 2-1でコピーしたInternal URL✅

DEFAULT_ADMIN_IDadmin （後で変更可）✅

TOKEN_EXPIRE_HOURS12✅
AUTO_CLEANUP_DAYS30（0で無効）—

SECRET_KEY の生成方法（ターミナルで実行）:

# Python で生成
python3 -c "import secrets; print(secrets.token_hex(32))"

# または openssl で生成
openssl rand -hex 32
💡 Render の「Generate Value」ボタンを使うと自動生成できます

Cloudflare R2 — 写真ストレージの設定（推奨）

4-1
なぜR2が必要か ▶
⚠ Renderの無料プランはディスクが「揮発性」です。再デプロイのたびにローカル保存した写真が消えます。本番運用には必ずR2またはS3を使ってください
Local (Render)Cloudflare R2
料金無料10GBまで無料
永続性❌ 再デプロイで消える✅ 永続
設定の手間不要10分程度
本番運用❌ 不可✅ 推奨
4-2
R2バケットを作成する ▶

① dash.cloudflare.com にログイン

② 左メニュー「R2 Object Storage」→「Create bucket」

項目値
Bucket namebuaizai-photos
LocationAsia Pacific (APAC)

③ 作成後、バケット設定 → 「Settings」→「Public Access」→「Allow Access」を有効化

④ 「Public bucket URL」をコピーしてメモ（例: https://pub-XXXX.r2.dev）

4-3
APIトークンを発行する ▶

① Cloudflareダッシュボード右上 → アイコン → 「My Profile」

② 「API Tokens」→「Create Token」→「Create Custom Token」

項目値
Token namebuaizai-r2
PermissionsAccount → R2 Storage → Edit
Account ResourcesInclude → All accounts

③「Continue to summary」→「Create Token」→ 表示されたトークンをコピー（一度しか表示されません）

④ R2の設定画面でアカウントIDをコピー（URLのdash.cloudflare.com/ACCOUNT_ID/r2の部分）

4-4
RenderにR2の環境変数を追加する ▶

Render → buaizai-api → Environment に以下を追加（または更新）:

KeyValue
STORAGE_BACKENDr2（localから変更）
R2_ACCOUNT_IDCloudflareのアカウントID
R2_ACCESS_KEY_ID作成したAPIトークンのAccess Key ID
R2_SECRET_ACCESS_KEY作成したAPIトークンのSecret
R2_BUCKET_NAMEbuaizai-photos
R2_PUBLIC_URLコピーしたPublic bucket URL

変数を保存すると自動で再デプロイされます。

💡 R2のAPIトークンには「Access Key ID」と「Secret Access Key」の2つがあります。R2バケット画面 → 「Manage R2 API tokens」から確認できます

ネットワークプリンター連携（オプション）

7-1
対応プリンターと構成 ▶

ZPL対応のラベルプリンター（Brother QL, Zebra ZD, DYMO など）をWi-Fiネットワーク上に配置し、ミドルウェアPCを経由して印刷します。

スマートフォン → GitHub Pages → Render API → ×（プリンターに直接は届かない） スマートフォン → GitHub Pages → ローカルミドルウェアPC → プリンター ✅
💡 プリンターとミドルウェアPCは同一のWi-Fiネットワーク上にある必要があります
7-2
ミドルウェアをセットアップする ▶

① ミドルウェアPC（Windows/Mac/Raspberry Pi）に Python をインストール

② 以下の printer_server.py を作成して実行:

pip install flask flask-cors
# printer_server.py
from flask import Flask, request, jsonify
from flask_cors import CORS
import socket

app = Flask(__name__)
CORS(app)  # GitHub Pagesからのリクエストを許可

PRINTER_IP   = "192.168.1.200"  # ← プリンターのIPアドレスに変更
PRINTER_PORT = 9100              # ZPL標準ポート

def build_zpl(code: str, name: str) -> str:
    """ZPLラベルフォーマット (62mm x 29mm)"""
    safe_name = name[:22].replace("^", " ")
    return (
        "^XA"
        "^PW490"          # ラベル幅 (62mm @ 203dpi)
        "^LL230"          # ラベル高さ (29mm @ 203dpi)
        f"^FO20,10^BQN,2,5^FDQA,{code}^FS"   # QRコード
        f"^FO200,20^A0N,30,30^FD{code}^FS"   # 管理コード
        f"^FO200,60^A0N,22,22^FD{safe_name}^FS"  # 製品名
        "^XZ"
    )

@app.route("/print", methods=["POST"])
def print_label():
    data = request.get_json()
    code = data.get("code", "")
    name = data.get("name", "")
    if not code:
        return jsonify({"ok": False, "error": "code is required"}), 400
    zpl = build_zpl(code, name)
    try:
        with socket.socket(socket.AF_INET, socket.SOCK_STREAM) as s:
            s.settimeout(5)
            s.connect((PRINTER_IP, PRINTER_PORT))
            s.sendall(zpl.encode("utf-8"))
        return jsonify({"ok": True, "code": code})
    except Exception as e:
        return jsonify({"ok": False, "error": str(e)}), 500

@app.route("/health", methods=["GET"])
def health():
    return jsonify({"ok": True, "printer": PRINTER_IP})

if __name__ == "__main__":
    print(f"プリンターサーバー起動: http://0.0.0.0:8888")
    print(f"プリンターIP: {PRINTER_IP}:{PRINTER_PORT}")
    app.run(host="0.0.0.0", port=8888, debug=False)

③ 起動:

python printer_server.py

④ ミドルウェアPCのローカルIPを確認:

# Windows
ipconfig | findstr "IPv4"

# Mac/Linux
ifconfig | grep "inet " | grep -v 127.0.0.1

⑤ Webアプリの「管理設定 → プリンターURL」に入力:

http://192.168.1.100:8888/print
⚠ スマートフォンとミドルウェアPCが同じWi-Fiに接続されている必要があります。社内LAN外からは使用できません

トラブルシューティング

❓
よくあるエラーと解決方法 ▶
症状原因解決方法
ログインできないDEFAULT_ADMIN_PW の設定ミスRenderの環境変数を確認。変更後はサービスを Manual Deploy
「CORS error」がコンソールに出るALLOWED_ORIGINS が未設定GitHub PagesのURLをALLOWED_ORIGINSに追加
写真が表示されない（再デプロイ後）Localストレージは揮発性Step 4 を参照してR2に移行する
APIが500エラーを返すDATABASE_URLが間違っているRender DBのInternal URLを再確認。External URLは不可
最初のアクセスが遅い（30秒）FreeプランのスリープStarter以上にアップグレード、またはcron jobで定期的にpingする
カメラが起動しないHTTPSが必要GitHub PagesはHTTPS。localhostでのテストはOK
QRが読み取れないカメラの解像度不足明るい場所で撮影。QRコードのサイズを大きくする
❓
Renderのログ確認方法 ▶

Render → buaizai-api → 「Logs」タブでリアルタイムログを確認できます。

# エラー例と意味
"relation 'items' does not exist"
→ DBの初期化が失敗。DATABASE_URLが正しいか確認

"password authentication failed"
→ DATABASE_URLのパスワードが間違っている

"ModuleNotFoundError: No module named 'xxx'"
→ requirements.txt に該当パッケージを追加してredeploy
